## Accomplishments

1. **Full IEC 61131-3 Ed.3 parser** with CODESYS OOP extensions, error recovery, CST-first architecture with trivia preservation
2. **Host-based unit testing** — the killer feature. TEST_CASE syntax, assertions, deterministic time, JUnit XML for CI
3. **Two-pass semantic analysis** with IEC type lattice, cross-file resolution, vendor-aware diagnostics
4. **Tree-walking interpreter** with scan-cycle semantics, full IEC standard library (10 FBs, 44+ functions)
5. **Multi-vendor emission** — write once, emit for Beckhoff/Schneider/Portable with round-trip stability
6. **Full LSP** with diagnostics, go-to-def, hover, completion, references, rename + VS Code extension
7. **MCP server** + Claude Code skills making this the first AI-native ST development environment

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Go language | Single binary, fast builds, agents write Go well |
| Interpreter only (no C++ transpiler) | Simpler architecture, good enough for testing |
| CST-first parser | Enables formatter + LSP without retrofit |
| Two-pass type checker (MATIEC-style) | Handles forward references and ANY type hierarchy |
| MIT license | Maximum permissiveness for adoption |

## Tech Debt (from audit)

- Preprocessor not wired into check/emit/fmt/lint/sim/test/LSP/MCP pipelines
- `stc sim` skips semantic analysis before execution
- Dead exports: `lint.Lint()`, `analyzer.AnalyzeFiles()`
- Skills don't reference MCP tool names
- Incremental caching only in `stc check`

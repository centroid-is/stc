## Cross-Phase Integration

### Critical: go vet Build Failures
- `pkg/parser/error.go:20` — `fmt.Sprintf` result passed as format arg to `Errorf`
- `pkg/checker/usage.go:90` — `fmt.Sprintf` result passed as format arg to `Warnf`

Fix: remove inner `fmt.Sprintf`, pass format string and args directly.

### High: Preprocessor Pipeline Gap
The preprocessor (`pkg/preprocess`) is only called from `stc pp`. All other commands (check, emit, fmt, lint, sim, test, LSP, MCP) call `parser.Parse()` directly on raw source. Files with `{IF defined()}` directives are parsed as literal text rather than preprocessed.

### Medium: Simulation Skips Semantic Analysis
`stc sim` parses and executes without running `analyzer.Analyze()`. Type-erroneous programs enter the interpreter silently.

### Low: Dead Exports
- `pkg/lint.Lint()` batch function unused
- `pkg/analyzer.AnalyzeFiles()` unused

### Low: Skills/MCP Documentation Gap
Skills document CLI commands but not MCP tool names.

    Create pkg/incremental/analyzer.go with:

    ```go
    type IncrStats struct {
        TotalFiles   int
        StaleFiles   int
        SkippedFiles int
    }

    type IncrementalAnalyzer struct {
        cache    *FileCache
        graph    *DepGraph
        table    *symbols.Table
        cacheDir string      // directory for .stc-cache/
        stats    IncrStats
    }
    ```

    - `func NewIncrementalAnalyzer(cacheDir string) *IncrementalAnalyzer` — creates analyzer, calls cache.LoadIndex(cacheDir) to restore previous hashes. Creates fresh DepGraph and Table.
    - `func (ia *IncrementalAnalyzer) Stats() IncrStats`
    - `func (ia *IncrementalAnalyzer) Analyze(filenames []string, cfg *project.Config) analyzer.AnalysisResult`

    The Analyze method implements this algorithm:
    1. Read all files from disk, compute ContentHash for each.
    2. Identify stale files: where cache.IsStale(filename, hash) returns true.
    3. Identify removed files: files in cache.Files() but not in filenames. Call graph.RemoveFile and table.PurgeFile for each.
    4. For stale files: parse them (parser.Parse), store in cache (cache.Store), scan for deps (graph.ScanFile).
    5. For non-stale files that NeedsParse: re-parse but don't count as stale (session-local re-parse since ASTs aren't persisted to disk).
    6. For non-stale files with cached ParseResult: load from cache, scan for deps (graph.ScanFile) to rebuild the in-memory dep graph.
    7. Compute dirty set: graph.AllDirty(staleFilenames) — this includes stale files and their transitive dependents.
    8. For each file in dirty set: call table.PurgeFile(filename).
    9. Collect all *ast.SourceFile from cache for the full file set (both dirty and clean).
    10. Run full analysis: use analyzer.Analyze(allFiles, cfg) — this rebuilds the symbol table from scratch. (This is the simple v1 approach: we always do a full semantic pass but skip parsing unchanged files. The dep graph identifies WHICH files were re-parsed, and the hash cache avoids re-reading/re-parsing unchanged files.)

    **Simplification note:** For v1, the main win is skipping parse for unchanged files (parsing is ~40% of total analysis time on large projects). Full semantic analysis still runs on all files because the symbol table is rebuilt from parsed ASTs. This is correct because semantic analysis needs the full program context. The dep graph is still useful: it enables future optimization where we only re-run semantic analysis on dirty files, and it's needed for the LSP integration where we want to know which open documents need diagnostics refreshed.

    After analysis completes:
    - Store stats (TotalFiles=len(filenames), StaleFiles=len(staleFiles), SkippedFiles=total-stale)
    - Call cache.SaveIndex(cacheDir) to persist hashes for next invocation

    Update analyzeDocument to do multi-file analysis when multiple documents are open:
    - Parse the changed document (as before)
    - If more than one document is open, collect all open documents' ParseResults and run analyzer.Analyze on ALL of them together. This gives cross-file symbol resolution.

    The LSP doesn't use the disk cache (it's a long-running process), but it benefits from cross-file analysis. The key change: instead of analyzing each document in isolation, analyze all open documents together so that go-to-definition and type checking work across files.

    Update the analyzeDocument method:
    ```go
    func (s *DocumentStore) analyzeDocument(doc *Document) {
        filename := uriToFilename(doc.URI)
        result := parser.Parse(filename, doc.Content)
        doc.ParseResult = &result

        // Collect all open documents for cross-file analysis
        s.mu.RLock()
        var allFiles []*ast.SourceFile
        for _, d := range s.docs {
            if d.ParseResult != nil {
                allFiles = append(allFiles, d.ParseResult.File)
            }
        }
        s.mu.RUnlock()

        // Include the current document (may not be in docs yet during Open)
        found := false
        for _, f := range allFiles {
            if f == result.File {
                found = true
                break
            }
        }
        if !found {
            allFiles = append(allFiles, result.File)
        }

        analysisResult := analyzer.Analyze(allFiles, nil)

        // Store full analysis result on all open documents
        s.mu.RLock()
        for _, d := range s.docs {
            d.AnalysisResult = &analysisResult
        }
        s.mu.RUnlock()
        doc.AnalysisResult = &analysisResult
    }
    ```

    This ensures cross-file features (go-to-def across files, FB instance type resolution) work correctly. Note: we store the SAME AnalysisResult reference on all docs since the symbol table is shared.

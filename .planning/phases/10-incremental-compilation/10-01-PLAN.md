    Create pkg/incremental/filecache.go with:
    - `func ContentHash(content []byte) string` — returns hex-encoded SHA-256 of content. Use crypto/sha256 and encoding/hex.
    - `type CacheEntry struct { Hash string; ParseResult *parser.ParseResult }` — in-memory entry
    - `type FileCache struct { entries map[string]*CacheEntry }` — keyed by filename
    - `func NewFileCache() *FileCache`
    - `func (c *FileCache) IsStale(filename, currentHash string) bool` — returns true if entry missing or hash differs
    - `func (c *FileCache) Store(filename, hash string, result *parser.ParseResult)` — stores or replaces entry
    - `func (c *FileCache) Load(filename string) (*parser.ParseResult, bool)` — returns cached ParseResult
    - `func (c *FileCache) Remove(filename string)` — removes entry
    - `func (c *FileCache) Files() []string` — returns all cached filenames (sorted)

    For on-disk persistence (the index only — ASTs stay in memory per invocation):
    - `type IndexEntry struct { Hash string` backtick`json:"hash"` backtick `}` — serializable
    - `type DiskIndex struct { Version int` backtick`json:"version"` backtick`; Files map[string]IndexEntry` backtick`json:"files"` backtick`}` — version=1
    - `func (c *FileCache) SaveIndex(dir string) error` — writes `.stc-cache/index.json` inside dir. Create `.stc-cache/` dir if needed. Marshal DiskIndex with current hashes. Use os.MkdirAll and os.WriteFile.
    - `func (c *FileCache) LoadIndex(dir string) error` — reads `.stc-cache/index.json`, populates entries with hash only (ParseResult nil — indicates "known hash but needs re-parse"). This means IsStale returns false for matching hashes even before a Store call, which is the desired behavior: we know the file hasn't changed, we just need to re-parse it on first access.
    - Add a `func (c *FileCache) NeedsParse(filename string) bool` — returns true if entry exists but ParseResult is nil (loaded from index, not yet parsed this session).

Create `pkg/emit/vendor.go` with:
- `Target` type (string enum): `TargetBeckhoff = "beckhoff"`, `TargetSchneider = "schneider"`, `TargetPortable = "portable"`
- `Options` struct with `Target Target`, `Indent string` (default "    " 4-space), `UppercaseKeywords bool` (default true)
- `LookupTarget(name string) Target` to normalize target names

Create `pkg/emit/emit.go` with:
- `Emit(file *ast.SourceFile, opts Options) string` — main entry point
- Internal `emitter` struct with `buf strings.Builder`, `indent int`, `opts Options`
- The emitter walks the AST using a type-switch approach (NOT the visitor pattern — direct switches are clearer for emission)
- Key methods: `emitDecl(d ast.Declaration)`, `emitStmt(s ast.Statement)`, `emitExpr(e ast.Expr)`, `emitTypeSpec(t ast.TypeSpec)`, `emitVarBlock(vb *ast.VarBlock)`, `emitTrivia(trivia []ast.Trivia)`

Vendor filtering logic:
- If target is Schneider: skip MethodDecl, PropertyDecl, InterfaceDecl entirely; skip PointerType and ReferenceType (emit a comment "// skipped: POINTER TO not supported by schneider")
- If target is Portable: same as Schneider PLUS skip 64-bit named types (LINT, LREAL, LWORD, ULINT) — emit a comment noting the skip
- For Beckhoff: emit everything as-is

Trivia handling:
- Before each node, emit its LeadingTrivia (whitespace + comments)
- After each node, emit its TrailingTrivia
- This preserves the original formatting and comments

Round-trip approach:
- The emitter does NOT try to reconstruct original whitespace from the AST trivia for round-trip stability. Instead it uses a canonical formatting: 4-space indent, uppercase keywords, newlines between declarations.
- Round-trip stability means: `emit(parse(src))` may differ from `src` in whitespace, BUT `emit(parse(emit(parse(src))))` MUST equal `emit(parse(src))` — i.e., the emitter's own output is stable when re-parsed and re-emitted.

Indentation:
- `emitIndent()` writes `opts.Indent * indent` spaces
- Each block (VAR..END_VAR, IF..END_IF, etc.) increases indent by 1

Keyword casing:
- All ST keywords emitted as UPPERCASE (IEC standard)
- Identifiers preserve original casing from the AST

Operator emission:
- For BinaryExpr, use Op.Text from the AST token (preserves original operator text like "AND", "OR", "+", ":=")
- For UnaryExpr, same approach

Semicolons:
- AssignStmt, CallStmt, ReturnStmt, ExitStmt, ContinueStmt, EmptyStmt all end with ";"
- VarDecl ends with ";"

Create `pkg/emit/emit_test.go` with:
- Unit tests for each node type
- Round-trip tests using parser.Parse -> emit.Emit -> parser.Parse -> emit.Emit -> compare
- Vendor-specific tests (Beckhoff emits OOP, Schneider strips it, Portable strips more)
- Test at least 3 comprehensive ST source strings covering: simple program, FB with methods, type declarations

Important: Use `github.com/centroid-is/stc/pkg/parser` in tests for round-trip verification. Import as needed.

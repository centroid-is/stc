    - ParseAddress("%QW8") returns IOAddress{Area:AreaOutput, Size:SizeWord, ByteOffset:8, BitOffset:0}

    - ParseAddress("%M*") returns IOAddress{Area:AreaMemory, IsWildcard:true}
    - ParseAddress("%ZZ0") returns error (invalid area)

    - IOTable.SetByte/GetByte on area Q, offset 4 round-trips correctly

    - IOTable.SetDWord/GetDWord on area M, offset 48 round-trips
    - IOTable.Read(addr) for bit addr returns Value{Kind:ValBool}
    - IOTable.Read(addr) for word addr returns Value{Kind:ValInt}
    - IOTable.Write(addr, BoolValue(true)) sets correct bit
    - IOTable.Write(addr, IntValue(0x1234)) for word addr sets correct bytes

  <action>
    Create `pkg/iomap/address.go`:
    - Define `type Area byte` with constants `AreaInput='I'`, `AreaOutput='Q'`, `AreaMemory='M'`.
    - Define `type Size byte` with constants `SizeBit='X'`, `SizeByte='B'`, `SizeWord='W'`, `SizeDWord='D'`.
    - Define `type IOAddress struct { Area Area; Size Size; ByteOffset int; BitOffset int; IsWildcard bool }`.
    - Add `func (a IOAddress) ByteSpan() (offset int, length int)` -- returns byte offset and byte count (1 for X/B, 2 for W, 4 for D).
    - Add `func (a IOAddress) String() string` -- canonical form like "%IX0.0", "%QW4".
    - Implement `func ParseAddress(s string) (IOAddress, error)`:
      - Input must start with `%`.
      - Next char is area: I, Q, M (case-insensitive). Error on anything else.
      - Next char is optional size: X, B, W, D (case-insensitive). If next char is `*` or digit, default to SizeBit for bit-style addresses (when there is a dot in the position) or SizeByte otherwise.
      - Actually, per IEC: if size is omitted and position contains a dot, it is a bit address (SizeBit). If size is omitted and position is just a number, it is a byte address.
      - If next char is `*`, set IsWildcard=true, done.
      - Otherwise parse byte offset as integer. If followed by `.`, parse bit offset (0-7). Bit offset only valid for SizeBit.
      - Validate: bit offset 0-7, byte offset >= 0. Return error on invalid.

    - `func NewIOTable() *IOTable` -- allocate I(1024), Q(1024), M(4096) byte slices.
    - Private `func (t *IOTable) area(a Area) []byte` returns the right slice.
    - Private `func (t *IOTable) ensureCapacity(a Area, needed int)` grows slice if needed.
    - `func (t *IOTable) GetBit(a Area, byteOff, bitOff int) bool`
    - `func (t *IOTable) SetBit(a Area, byteOff, bitOff int, v bool)`
    - `func (t *IOTable) GetByte(a Area, off int) byte`
    - `func (t *IOTable) SetByte(a Area, off int, v byte)`
    - `func (t *IOTable) GetWord(a Area, off int) uint16` -- little-endian
    - `func (t *IOTable) SetWord(a Area, off int, v uint16)` -- little-endian
    - `func (t *IOTable) GetDWord(a Area, off int) uint32` -- little-endian
    - `func (t *IOTable) SetDWord(a Area, off int, v uint32)` -- little-endian
    - `func (t *IOTable) Read(addr IOAddress) Value` -- dispatches by Size to Get* methods, returns interp.Value. Import cycle concern: do NOT import interp. Instead, return raw interface{} or define a generic read that returns (uint32, bool). BETTER: keep IOTable purely in terms of bytes and let the caller (ScanCycleEngine) convert to Value. So Read/Write should NOT be on IOTable. IOTable provides only Get/Set typed methods. The ScanCycleEngine will call GetBit/SetBit/GetWord/SetWord etc. based on the IOAddress.Size.
    - CORRECTION: Do NOT add Read/Write that return Value to IOTable. Keep IOTable as pure byte-level storage. Integration with Value happens in Plan 02.
    - `func (t *IOTable) Reset()` -- zeroes all three arrays.

  <name>Task 1: SimulationEngine with closed-loop waveform + plant model integration</name>
  <files>pkg/sim/engine.go, pkg/sim/result.go, pkg/sim/engine_test.go</files>
  <behavior>
    - SimulationEngine runs N cycles at fixed dt, recording outputs each cycle
    - Waveforms inject values into named program inputs before each scan
    - After each scan, program outputs feed into plant models
    - Plant model outputs feed back as program inputs on the next cycle
    - Running same config twice produces identical CycleRecord sequences
    - Engine returns SimResult with all cycle records and final output snapshot
    - Test: Motor program (RUN input from step waveform -> motor plant -> SPEED output increases over time)
    - Test: Determinism — two runs produce identical float64 results
  </behavior>
  <action>
    Create `pkg/sim/result.go`:

    ```go
    type CycleRecord struct {
        Cycle   int                     `json:"cycle"`
        Time    time.Duration           `json:"time"`
        Inputs  map[string]interp.Value `json:"inputs"`
        Outputs map[string]interp.Value `json:"outputs"`
    }

    type SimResult struct {
        Cycles   []CycleRecord         `json:"cycles"`
        Duration time.Duration         `json:"duration"`
        NumCycles int                  `json:"num_cycles"`
    }
    ```

    Create `pkg/sim/engine.go`:

    Define `WaveformBinding` struct: `InputName string`, `Generator *WaveformGenerator`. Maps a waveform to a program input name.

    Define `PlantBinding` struct: `Model PlantModel`, `OutputToPlant map[string]string` (program output name -> plant input name), `PlantToInput map[string]string` (plant output name -> program input name). This is the closed-loop wiring.

    Define `SimConfig` struct:
    - Program *ast.ProgramDecl
    - NumCycles int (how many scan cycles to run)
    - CycleDt time.Duration (time step per cycle, e.g. 10ms)
    - Waveforms []WaveformBinding (sensor injections)
    - Plants []PlantBinding (closed-loop models)
    - RecordInterval int (record every N cycles, default 1 — for large sims to reduce memory)

    Define `SimulationEngine` struct holding config, a `*interp.ScanCycleEngine`, and accumulated `[]CycleRecord`.

    `NewSimulationEngine(cfg SimConfig) *SimulationEngine` — creates the scan cycle engine from cfg.Program, stores config.

    `func (s *SimulationEngine) Run() (*SimResult, error)`:
    1. For cycle = 0 to NumCycles-1:
       a. t = cycle * CycleDt (virtual time for this cycle)
       b. For each WaveformBinding: val = generator.Generate(t), engine.SetInput(binding.InputName, val)
       c. engine.Tick(CycleDt)
       d. Collect program outputs into a map
       e. For each PlantBinding: build plant inputs from program outputs using OutputToPlant mapping, call plant.Update(plantInputs, CycleDt), feed plant outputs back via PlantToInput as engine.SetInput for next cycle
       f. If cycle % RecordInterval == 0: snapshot inputs and outputs into CycleRecord
    2. Return SimResult with all recorded cycles

    Note: Plant outputs are fed back as inputs on the NEXT cycle (not same cycle). This matches real PLC behavior where inputs are read at scan start.

    RecordInterval defaults to 1 if 0.

    Snapshot function: iterate over output names from GetOutput, copy values into map. Inputs: collect from waveform values + plant feedback values.

1. Create `pkg/parser/stmt.go`:
   - `parseStatements(stop ...lexer.TokenKind) []ast.Statement` — parses statements until a stop token or EOF
   - `parseStatement() ast.Statement` — dispatches based on current token:
     - KwIf → `parseIfStmt()`
     - KwCase → `parseCaseStmt()`
     - KwFor → `parseForStmt()`
     - KwWhile → `parseWhileStmt()`
     - KwRepeat → `parseRepeatStmt()`
     - KwReturn → `&ast.ReturnStmt{}`, expect optional semicolon
     - KwExit → `&ast.ExitStmt{}`, expect optional semicolon
     - KwContinue → `&ast.ContinueStmt{}`, expect optional semicolon
     - Semicolon → `&ast.EmptyStmt{}` (bare semicolon)
     - Ident → could be assignment (`x := expr;`) or FB call (`fb(args);`) — parse LHS expr, then:
       - If `:=` follows → AssignStmt
       - If `(` follows and LHS is Ident/MemberAccess → CallStmt with named args
       - Otherwise → ExprStmt or error
     - Otherwise → recoverStatement()

   - `parseIfStmt() *ast.IfStmt`:
     - Expect KwIf, parse condition expr, expect KwThen
     - Parse body statements until KwElsif, KwElse, or KwEndIf
     - Parse zero or more ELSIF branches
     - Optional ELSE branch
     - Expect KwEndIf, optional semicolon

   - `parseCaseStmt() *ast.CaseStmt`:
     - Expect KwCase, parse selector expr, expect KwOf
     - Parse branches: each branch is comma-separated labels (value, range `expr..expr`), expect Colon, parse body until next label or KwElse or KwEndCase
     - Optional ELSE branch
     - Expect KwEndCase, optional semicolon

   - `parseForStmt() *ast.ForStmt`:
     - Expect KwFor, parse variable (Ident), expect Assign, parse from expr, expect KwTo, parse to expr
     - Optional KwBy + step expr
     - Expect KwDo, parse body, expect KwEndFor, optional semicolon

   - `parseWhileStmt() *ast.WhileStmt`:
     - Expect KwWhile, parse condition, expect KwDo, parse body, expect KwEndWhile, optional semicolon

   - `parseRepeatStmt() *ast.RepeatStmt`:
     - Expect KwRepeat, parse body, expect KwUntil, parse condition, expect Semicolon

   - `parseCallArgs() []*ast.CallArg`:
     - Parse named arguments: `Name := Expr` or `Name => Expr` (for outputs), comma-separated
     - Also handle positional arguments (just Expr without name)

2. Create `pkg/parser/expr.go` — Pratt expression parser:
   - `func (p *Parser) parseExpr(minPrec int) ast.Expr` — the core Pratt loop
   - `func (p *Parser) parseUnaryExpr() ast.Expr`:
     - KwNot → UnaryExpr{Op: NOT, Operand: parseUnaryExpr()}
     - Minus → UnaryExpr{Op: MINUS, Operand: parseUnaryExpr()}
     - Otherwise → parsePrimaryExpr()
   - `func (p *Parser) parsePrimaryExpr() ast.Expr`:
     - IntLiteral, RealLiteral, StringLiteral, etc. → Literal node
     - TimeLiteral, DateLiteral, etc. → Literal node with appropriate LiteralKind
     - TypedLiteral → Literal node with TypePrefix
     - KwTrue, KwFalse → Literal node (LitBool)
     - Ident → Ident node, then check for postfix:
       - `(` → CallExpr (function call in expression position)
       - `.` → MemberAccessExpr, then continue checking for more postfix
       - `[` → IndexExpr
       - `^` → DerefExpr
     - LParen → ParenExpr

   - Precedence table (from RESEARCH.md):
     ```
     prec 1: OR
     prec 2: XOR
     prec 3: AND, &
     prec 4: =, <>
     prec 5: <, >, <=, >=
     prec 6: +, -
     prec 7: *, /, MOD
     prec 8: ** (right-associative)
     ```
   - `func (p *Parser) infixPrecedence(kind lexer.TokenKind) int` — returns precedence for binary operators, 0 for non-operators
   - In the Pratt loop: parse left with `parseUnaryExpr()`, then loop: if next token has precedence >= minPrec, consume operator, parse right with `parseExpr(prec+1)` (or `prec` for right-associative `**`), wrap in BinaryExpr

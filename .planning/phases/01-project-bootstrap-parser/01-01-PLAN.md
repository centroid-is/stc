1. Create `pkg/source/source.go`:
   - `type Pos struct { File string; Line int; Col int; Offset int }` with JSON tags
   - `func (p Pos) String() string` returning `"file:line:col"` format
   - `type Span struct { Start Pos; End Pos }` with JSON tags
   - `func SpanFrom(start, end Pos) Span` constructor
   - `type SourceFile struct` with fields: `Name string`, `Content string`, `lines []int` (line start offsets, computed lazily)
   - `func New(name, content string) *SourceFile` constructor
   - `func (sf *SourceFile) PosFromOffset(offset int) Pos` — converts byte offset to line:col using binary search on line offsets
   - `func (sf *SourceFile) LineContent(line int) string` — returns content of a line for diagnostic display
   - Internal `computeLines()` method that scans for newlines and builds the offset table

2. Create `pkg/source/source_test.go`:
   - TestPosString: verify `Pos{File: "test.st", Line: 10, Col: 5}.String()` equals `"test.st:10:5"`
   - TestSourceFilePosFromOffset: create SourceFile with multiline content, verify offset-to-pos conversion for first char, middle of line 2, start of line 3
   - TestSourceFileLineContent: verify line extraction by index

3. Create `pkg/diag/diagnostic.go`:
   - `type Severity int` with constants: `Error = iota`, `Warning`, `Info`, `Hint`
   - `func (s Severity) String() string` returning "error", "warning", "info", "hint"
   - `func (s Severity) MarshalJSON() ([]byte, error)` returning the string value
   - `type Diagnostic struct` with fields: `Severity Severity`, `Pos source.Pos`, `EndPos source.Pos`, `Code string`, `Message string` — all with json tags
   - `func (d Diagnostic) String() string` returning `"file:line:col: severity: message"` (per CLI-05)
   - Import path: `github.com/centroid-is/stc/pkg/source`

4. Create `pkg/diag/collector.go`:
   - `type Collector struct { diags []Diagnostic }`
   - `func NewCollector() *Collector`
   - `func (c *Collector) Add(sev Severity, pos source.Pos, endPos source.Pos, code, msg string)`
   - `func (c *Collector) Errorf(pos source.Pos, code, format string, args ...any)` — convenience for errors
   - `func (c *Collector) Warnf(pos source.Pos, code, format string, args ...any)` — convenience for warnings
   - `func (c *Collector) All() []Diagnostic`
   - `func (c *Collector) HasErrors() bool`
   - `func (c *Collector) Errors() []Diagnostic` — filter to errors only

5. Create `pkg/diag/diagnostic_test.go`:
   - TestDiagnosticString: verify `Diagnostic{Severity: Error, Pos: source.Pos{File: "test.st", Line: 3, Col: 5}, Message: "unexpected token"}.String()` equals `"test.st:3:5: error: unexpected token"`
   - TestCollector: add multiple diagnostics, verify All() count, HasErrors(), Errors() filtering
   - TestDiagnosticJSON: marshal Diagnostic to JSON, verify `"severity":"error"` and `"pos"` fields present

6. Create `pkg/project/config.go`:
   - `type Config struct { Project ProjectConfig; Build BuildConfig; Lint LintConfig }` with toml tags
   - `type ProjectConfig struct { Name string; Version string }` with toml tags
   - `type BuildConfig struct { SourceRoots []string; VendorTarget string; LibraryPaths map[string]string }` with toml tags
   - `type LintConfig struct { NamingConvention string }` with toml tag
   - `func LoadConfig(path string) (*Config, error)` using `github.com/BurntSushi/toml`
   - `func FindConfig(dir string) (string, error)` — walks up from dir looking for stc.toml

1. Create `pkg/lexer/lexer.go`:
   - `type Lexer struct` with fields: `src string`, `file string`, `pos int` (current byte offset), `line int`, `col int`, `tokens []Token`
   - `func New(filename, source string) *Lexer` constructor
   - `func (l *Lexer) Tokenize() []Token` — scans all tokens including trivia, returns complete token list
   - `func Tokenize(filename, source string) []Token` — convenience wrapper
   - Core scanning loop in `scan() Token`:
     - Skip nothing — whitespace becomes Whitespace tokens (CST fidelity)
     - Identify next character and dispatch to appropriate scanner:

   **Whitespace scanner:** Consume runs of space/tab/CR/LF, emit `Whitespace` token

   **Line comment scanner:** When `//` seen, consume to end of line, emit `LineComment` token

   **Block comment scanner:** When `(*` seen, track nesting depth (increment on `(*`, decrement on `*)`), emit `BlockComment` token. MUST handle nested comments per IEC spec.

   **Pragma scanner:** When `{` seen, consume until matching `}`, emit `Pragma` token. Content is raw text including braces.

   **String literal scanner:** When `'` seen, consume until closing `'`, handle `''` escape for embedded single quotes, emit `StringLiteral`. When `"` seen, same but for wide strings, emit `WStringLiteral`.

   **Number/literal scanner:** Handles these forms:
     - Simple integer: `42`, `0`
     - Multi-base integer: `16#FF`, `8#77`, `2#1010`, `16#DEAD_BEEF` (underscores in hex)
     - Real: `3.14`, `1.0E-5`, `1.5e+3`
     - Time literals: `T#5s`, `T#1h30m`, `TIME#100ms`, `t#1d2h3m4s5ms` — emit `TimeLiteral`
     - Date literals: `D#2024-01-15`, `DATE#2024-01-15` — emit `DateLiteral`
     - DateTime literals: `DT#2024-01-15-12:30:00` — emit `DateTimeLiteral`
     - TOD literals: `TOD#12:30:00` — emit `TodLiteral`
     - Typed literals: `INT#5`, `BOOL#TRUE`, `REAL#3.14` — emit `TypedLiteral`
     - Key insight: When scanning an identifier that could be a type prefix (INT, REAL, BOOL, etc.) and the next char is `#`, treat the whole thing as a typed literal. Similarly for T, TIME, D, DATE, DT, TOD followed by `#`.

   **Identifier/keyword scanner:** Scan `[a-zA-Z_][a-zA-Z0-9_]*`, then call `LookupKeyword()` to determine if it is a keyword. If keyword, emit the keyword token kind. Otherwise emit `Ident`. Preserve original casing in Token.Text.

   **Operator/punctuation scanner:** Match single and multi-character operators:
     - `:=` → Assign, `:` → Colon
     - `**` → Power, `*` → Star
     - `<=` → LessEq, `<>` → NotEq, `<` → Less
     - `>=` → GreaterEq, `>` → Greater
     - `=>` → Arrow, `=` → Eq
     - `..` → DotDot, `.` → Dot
     - All single-char: `+`, `-`, `/`, `(`, `)`, `[`, `]`, `,`, `;`, `^`, `#`, `&`

   **Illegal token:** Any unrecognized character emits `Illegal` with that character

   - Position tracking: maintain `line` (1-based) and `col` (1-based). Increment `col` per byte. On `\n`, increment `line`, reset `col` to 1. The `Pos` in each token is captured at scan start.

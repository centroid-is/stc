       ```go

           Failed   int           `json:"failed"`

           Warnings []string      `json:"warnings,omitempty"`
           Duration time.Duration `json:"duration_ns"`
       }
       ```

    2. In pkg/testing/runner.go, add RunOpts and RunWithOpts:
       ```go
       type RunOpts struct {
           LibraryFiles []*ast.SourceFile
           MockFiles    []*ast.SourceFile
       }

       func RunWithOpts(dir string, opts RunOpts) (*RunResult, error)
       ```
       Keep existing `Run(dir)` as a wrapper calling `RunWithOpts(dir, RunOpts{})`.

    3. In RunWithOpts, pass MockFiles and LibraryFiles into the test execution pipeline:
       - The fileContext already collects FunctionBlockDecl from test files
       - Add library FB declarations to fileContext.fbDecls (but only those WITHOUT bodies -- stubs)
       - Add mock FB declarations to fileContext.fbDecls (these HAVE bodies, override stubs)
       - Track which library FB types were used but have no mock and no body -- these are auto-stubs

    4. Auto-stub detection and warnings:
       - After test execution, for each FB instance that was created from a library stub (no body),
         add a warning: "auto-stub: FB type 'MC_MoveAbsolute' has no mock implementation; outputs are zero-valued"
       - Collect these in RunResult.Warnings
       - Detection approach: when initializeTestEnv encounters an FB type that is in library stubs
         but NOT in mock files and NOT in stdlib, it creates a zero-value FBInstance (NewUserFBInstance
         with a body-less decl already produces this behavior). Track the type name.

    5. I/O injection for IO-04:
       - In executeTestCase, register SET_IO and GET_IO as assertion-style functions:
         - SET_IO(area_str, byte_offset, bit_offset, value) -- sets a bit/word in IOTable
         - GET_IO(area_str, byte_offset, bit_offset) -- reads from IOTable
       - The IOTable is already created in ScanCycleEngine. For test cases that don't use
         ScanCycleEngine directly, create a standalone IOTable in the test env.
       - Register these via interpreter.RegisterFunction:
         ```go
         interpreter.RegisterFunction("SET_IO", func(args []interp.Value, pos ast.Pos) (interp.Value, error) {
             // args: area string ("I"/"Q"/"M"), byteOff int, bitOff int, value bool
             // Map to ioTable.SetBit(area, byteOff, bitOff, value)
             return interp.Value{}, nil
         })
         ```

    6. Write tests in pkg/testing/runner_test.go:
       - Test with a mock FB file that returns non-zero value, verify test sees mock behavior
       - Test with a library stub only (no mock), verify outputs are zero
       - Test that auto-stub warning appears in result.Warnings
       - Test SET_IO/GET_IO functions work in test cases

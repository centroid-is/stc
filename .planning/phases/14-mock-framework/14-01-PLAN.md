    - Test: MockFiles do NOT override user-defined symbols (IsLibrary=false) -- redeclaration error
    - Test: ValidateMockSignatures returns nil when mock FB has same inputs/outputs as stub
    - Test: ValidateMockSignatures returns error when mock has different parameter count
    - Test: ValidateMockSignatures returns error when mock has different parameter types
    - Test: ValidateMockSignatures skips FBs not found in library (no validation needed)
  </behavior>
  <action>
    1. In pkg/checker/resolve.go, add MockFiles field to ResolveOpts:
       ```go
       type ResolveOpts struct {
           LibraryFiles []*ast.SourceFile
           MockFiles    []*ast.SourceFile
       }
       ```

    2. In CollectDeclarations, after registering user files, register mock files.
       Mock files should override library symbols (IsLibrary=true) the same way user code does:
       - If existing symbol has IsLibrary=true, remove it and register mock
       - If existing symbol has IsLibrary=false (user code), emit redeclaration error
       Mock symbols should NOT be marked IsLibrary (they have bodies, they are real implementations).

       Add after the user files loop:
       ```go
       // Register mock files (highest priority, override library symbols)
       if len(opts) > 0 && opts[0].MockFiles != nil {
           for _, mockFile := range opts[0].MockFiles {
               r.collectFileDeclarations(mockFile, false) // false = not library
           }
       }
       ```
       This works because collectFileDeclarations already handles the IsLibrary override logic:
       when isLibrary=false and existing.IsLibrary=true, it removes the library entry and registers the new one.

    3. In pkg/vendor/mock.go, add ValidateMockSignatures function:
       ```go
       func ValidateMockSignatures(mockFiles []*ast.SourceFile, table *symbols.Table) []error
       ```
       - Walk mockFiles, find all FunctionBlockDecl declarations
       - For each, look up the same name in the symbol table
       - If found and it has a types.FunctionBlockType, compare:
         a. Input count and types must match
         b. Output count and types must match
         c. InOut count and types must match
       - Return slice of errors describing mismatches
       - Use strings.EqualFold for parameter name comparison
       - Compare types using Type.String() equality (same approach as checker)

    4. Add tests to pkg/checker/resolve_test.go for mock override semantics.
    5. Add tests to pkg/vendor/mock_test.go for ValidateMockSignatures.

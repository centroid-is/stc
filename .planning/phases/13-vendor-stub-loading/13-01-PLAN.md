<task type="auto" tdd="true">
  <name>Task 2: Modify Resolver to accept library files and mark library symbols</name>
  <files>pkg/checker/resolve.go, pkg/checker/resolve_test.go</files>
  <behavior>
    - Test: CollectDeclarations with LibraryFiles registers FB types that user code can reference
    - Test: Library symbols have IsLibrary=true in the symbol table
    - Test: User code declaring same-name FB as library does NOT produce redeclaration error (user overrides library)
    - Test: Library FB inputs/outputs are correctly typed so parameter validation works
    - Test: Two library files with same FB name -- second is silently ignored (first library wins)
  </behavior>
  <action>
1. Add a ResolveOpts struct to pkg/checker/resolve.go:
```go
type ResolveOpts struct {
    LibraryFiles []*ast.SourceFile // Parsed library stub files (registered before user code)
}
```

2. Change CollectDeclarations signature to:
```go
func (r *Resolver) CollectDeclarations(files []*ast.SourceFile, opts ...ResolveOpts)
```
The variadic opts pattern preserves backward compatibility -- existing callers pass no opts.

3. In CollectDeclarations, before the existing loop over `files`, add:
```go
if len(opts) > 0 && opts[0].LibraryFiles != nil {
    for _, libFile := range opts[0].LibraryFiles {
        r.collectFileDeclarations(libFile, true) // isLibrary=true
    }
}
```
Then call `r.collectFileDeclarations(file, false)` for user files.

4. Extract the existing per-file declaration logic into a helper `collectFileDeclarations(file *ast.SourceFile, isLibrary bool)`. In each resolve* method (resolveProgram, resolveFunctionBlock, resolveFunction, resolveTypeDecl, resolveInterface), after creating the Symbol and inserting it, set `sym.IsLibrary = isLibrary`.

5. For library symbols: when isLibrary=true and a redeclaration check finds an existing symbol, check if existing.IsLibrary is true. If so, silently skip (first library wins). If existing.IsLibrary is false, that means user code was registered first (shouldn't happen with correct ordering, but defensive). The key change: when isLibrary=false (user code) and existing.IsLibrary=true, REPLACE the library symbol with the user symbol instead of emitting a redeclaration error. This enables user code and later mock code to override library stubs.

6. To replace a library symbol with a user symbol: delete the existing symbol from global scope and re-register. Since Scope uses a map keyed by upper-case name, this is: `delete(scope.symbols, strings.ToUpper(name))` then re-insert.

7. Write tests in pkg/checker/resolve_test.go that create minimal AST SourceFiles (use parser to parse test strings), call CollectDeclarations with LibraryFiles, and verify:
   - Library FB types resolve from user code references
   - IsLibrary is set correctly
   - User FB overrides library FB without error
   - Input/output parameters are correctly registered on the FunctionBlockType

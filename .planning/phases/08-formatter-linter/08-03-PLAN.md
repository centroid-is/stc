<objective>
Close FMT-03 gap: parser trivia attachment so that parse -> format round-trips preserve all comments.

Purpose: The formatter correctly emits trivia when present on AST nodes, but the parser discards trivia tokens instead of attaching them. This means `stc fmt` silently drops all comments. This plan adds a post-parse trivia attachment pass to the parser.

Output: Modified parser that attaches LineComment and BlockComment tokens as LeadingTrivia/TrailingTrivia on the nearest AST nodes, plus end-to-end tests proving comments survive formatting.
</objective>

    Create pkg/parser/trivia.go with a post-parse trivia attachment function `attachTrivia(file *ast.SourceFile, allTokens []lexer.Token)`.

    Algorithm — walk allTokens in order, maintaining a "pending leading trivia" buffer:

    1. Build a flat list of all AST nodes by walking the tree (depth-first, collecting every Node with a NodeBase). Sort by Span.Start.Offset.

    2. Walk allTokens left-to-right. Maintain a `pendingTrivia []ast.Trivia` buffer for comment tokens only (skip Whitespace trivia — the formatter regenerates whitespace from structure). For each token:
       - If token is LineComment or BlockComment: convert to ast.Trivia, append to pendingTrivia buffer
       - If token is Whitespace: skip (do not attach — formatter handles whitespace)
       - If token is a non-trivia token: find the AST node whose span starts at or contains this token's offset. Flush pendingTrivia as LeadingTrivia to that node.

    3. For trailing trivia: a comment is "trailing" if it appears on the same line as the end of a node, with no intervening newline. After the main pass, scan for comment tokens that sit between a node's end offset and the next newline — reassign from the next node's LeadingTrivia to the previous node's TrailingTrivia.

    Simpler concrete approach for step 2-3:
    - Iterate through allTokens. Track which non-trivia token index we are at (correlate with p.tokens by offset matching).
    - For each non-trivia token, collect all preceding trivia since the last non-trivia token.
    - Comments on the SAME LINE as the PREVIOUS non-trivia token become TrailingTrivia of the node owning that previous token.
    - Remaining comments become LeadingTrivia of the node owning the current non-trivia token.
    - To map non-trivia tokens to AST nodes: build a map from token start offset -> *ast.NodeBase for each node's first token.

    Hook into parser.go Parse() function: after `file := p.parseSourceFile()`, call `attachTrivia(file, allTokens)` before returning.

    Do NOT change the trivia filtering loop in Parse() — keep the existing tokens/trivia split for the parser to work correctly. The attachment is a post-parse step.

    Create pkg/parser/trivia_test.go with tests per the behavior block above. Each test calls parser.Parse() and inspects LeadingTrivia/TrailingTrivia on specific AST nodes.

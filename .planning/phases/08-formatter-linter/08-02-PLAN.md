  <files>pkg/lint/lint.go, pkg/lint/rules.go, pkg/lint/naming.go, pkg/lint/plcopen.go, pkg/lint/lint_test.go</files>
  <behavior>
    - Test: Lint detects magic numbers in expressions (literal integers other than 0, 1, -1 in non-constant non-init contexts)
    - Test: Lint detects deeply nested control structures (default threshold: 3 levels of IF/FOR/WHILE/REPEAT/CASE)
    - Test: Lint detects long POU bodies (default threshold: 200 statements)
    - Test: Lint detects missing return type on FUNCTION (PLCopen guideline)
    - Test: Naming rule detects FB name not in PascalCase (e.g., "my_motor" flagged, "MyMotor" passes)
    - Test: Naming rule detects PROGRAM name not in PascalCase (e.g., "main_program" flagged)
    - Test: Naming rule detects variable names not in lower_snake_case (e.g., "MyVar" flagged, "my_var" passes)
    - Test: Naming convention is configurable: "none" disables all naming checks
    - Test: Lint returns empty diagnostics for clean, well-named code
    - Test: Lint diagnostics have correct file:line:col positions
    - Test: Each diagnostic has a LINT-prefixed code (LINT001, LINT002, etc.)
  </behavior>
  <action>
    Create `pkg/lint/rules.go`:
    - Define `Rule` interface: `Name() string`, `Check(file *ast.SourceFile, opts LintOptions) []diag.Diagnostic`.
    - Define `LintOptions` struct: `NamingConvention string` (values: "plcopen" default, "none" to disable naming).
    - Define `DefaultLintOptions()` returning `LintOptions{NamingConvention: "plcopen"}`.
    - Define lint diagnostic codes as constants:
      - `CodeMagicNumber = "LINT001"` — magic number in expression
      - `CodeDeepNesting = "LINT002"` — deeply nested control flow (>3 levels)
      - `CodeLongPOU = "LINT003"` — POU body exceeds 200 statements
      - `CodeMissingReturnType = "LINT004"` — FUNCTION without return type
      - `CodeNamingPOU = "LINT005"` — POU name not PascalCase
      - `CodeNamingVar = "LINT006"` — variable name not lower_snake_case
      - `CodeNamingConstant = "LINT007"` — constant not UPPER_SNAKE_CASE

    Create `pkg/lint/plcopen.go`:
    - Implement PLCopen rules as functions that return `[]diag.Diagnostic`:
      - `checkMagicNumbers(file *ast.SourceFile) []diag.Diagnostic`: walk all expressions in statement bodies. Flag integer/real literals that are not 0, 1, or -1, and are not in VAR CONSTANT init values or array range bounds. Use diag.Warning severity.
      - `checkNestingDepth(file *ast.SourceFile, maxDepth int) []diag.Diagnostic`: walk statements, track nesting depth for IF/FOR/WHILE/REPEAT/CASE. Flag when depth > maxDepth (default 3).
      - `checkPOULength(file *ast.SourceFile, maxStmts int) []diag.Diagnostic`: count body statements in each POU. Flag when > maxStmts (default 200).
      - `checkMissingReturnType(file *ast.SourceFile) []diag.Diagnostic`: for each FunctionDecl, if ReturnType is nil, emit warning.
    - All diagnostics use `source.Pos` derived from the node's `Span().Start` (same pattern as checker/vendor.go `spanPos`).

    Create `pkg/lint/naming.go`:
    - `checkNaming(file *ast.SourceFile, convention string) []diag.Diagnostic`:
      - If convention is "none", return nil immediately.
      - For convention "plcopen" (default):
        - POU names (ProgramDecl, FunctionBlockDecl, FunctionDecl): must be PascalCase. Regex: `^[A-Z][a-zA-Z0-9]*(_[A-Z][a-zA-Z0-9]*)*$` (allows FB_Motor style too — PascalCase with underscores between words where each segment starts uppercase).
        - Variable names (VarDecl.Names): must be lower_snake_case or lowerCamelCase. Regex: `^[a-z][a-z0-9]*(_[a-z0-9]+)*$` OR `^[a-z][a-zA-Z0-9]*$`. Simplify to: first char lowercase. Flag if first char is uppercase.
        - Constants (VarBlock.IsConstant): should be UPPER_SNAKE_CASE. Regex: `^[A-Z][A-Z0-9]*(_[A-Z0-9]+)*$`.
    - Helper: `isPascalCase(s string) bool`, `isLowerStart(s string) bool`, `isUpperSnake(s string) bool`.

    Create `pkg/lint/lint.go`:
    - Define `LintResult` struct: `Diags []diag.Diagnostic`.
    - `Lint(files []*ast.SourceFile, opts LintOptions) LintResult`:
      1. For each file, run PLCopen rules (magic numbers, nesting, POU length, missing return type).
      2. For each file, run naming rules with opts.NamingConvention.
      3. Collect all diagnostics, sort by file then line then col.
      4. Return LintResult.
    - `LintFile(file *ast.SourceFile, opts LintOptions) LintResult` convenience for single file.

    Create `pkg/lint/lint_test.go`:
    - Test helper: `lintST(code string) []diag.Diagnostic` that parses then lints with default options.
    - Test each rule independently with targeted ST snippets.
    - Test clean code produces zero diagnostics.
    - Test that diagnostic codes match expected LINT0xx pattern.
    - Test naming convention "none" disables naming checks.
  </action>

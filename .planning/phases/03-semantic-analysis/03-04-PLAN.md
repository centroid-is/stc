<task type="auto" tdd="true">
  <name>Task 2: Unused variable detection and unreachable code detection</name>
  <files>pkg/checker/usage.go, pkg/checker/usage_test.go, pkg/checker/testdata/unused_var.st, pkg/checker/testdata/unreachable.st</files>
  <read_first>
    - pkg/symbols/scope.go (Scope.Symbols() for iterating declared symbols)
    - pkg/symbols/symbol.go (Symbol.Used flag, Symbol.Kind)
    - pkg/ast/stmt.go (ReturnStmt, ExitStmt — after these, code is unreachable)
    - pkg/checker/diag_codes.go (SEMA012, SEMA013 codes)
  </read_first>
  <behavior>
    - TestUnusedVarDetected: PROGRAM with declared but unused variable emits SEMA012 warning
    - TestUsedVarNoWarning: PROGRAM with declared and used variable emits no SEMA012
    - TestUnusedVarInput: VAR_INPUT variables that are unused should NOT be warned (they are interface points)
    - TestUnusedVarOutput: VAR_OUTPUT variables that are unused should NOT be warned
    - TestUnreachableAfterReturn: Code after RETURN statement emits SEMA013 warning
    - TestUnreachableAfterExit: Code after EXIT statement in a loop emits SEMA013 warning
    - TestNoUnreachableIfLastStmt: RETURN as last statement in body emits no warning (nothing after it)
  </behavior>
  <action>
    Create `pkg/checker/usage.go`:
    - CheckUsage(files []*ast.SourceFile, table *symbols.Table, diags *diag.Collector):
      1. Unused variable check: Walk all scopes in the table. For each Symbol with Kind=KindVariable and Used=false:
         - Skip if the variable is in a VAR_INPUT, VAR_OUTPUT, or VAR_IN_OUT block (these are interface points, expected to be set/read externally)
         - Skip if the variable is in a VAR_GLOBAL block (may be used by other POUs)
         - Emit SEMA012 Warning at the symbol's declaration position
         - Message: "variable '%s' is declared but never used"
      2. Unreachable code check: Walk all statement lists in all POUs. For each statement list:
         - If a ReturnStmt or ExitStmt is found and it is NOT the last statement, emit SEMA013 Warning for each subsequent statement
         - Position: use the span of the first unreachable statement
         - Message: "unreachable code after %s statement" (RETURN or EXIT)

    Create test fixtures:
    - testdata/unused_var.st: PROGRAM with unused local VAR, used VAR_INPUT
    - testdata/unreachable.st: PROGRAM with RETURN followed by assignment

    NOTE: The usage checker depends on Symbol.Used being set by the type checker (Plan 03). When running standalone tests, set up symbols manually. When integrated via analyzer (Plan 05), the checker marks usage first, then usage.go reads it.
  </action>
  <verify>
    <automated>cd /Users/jonb/Projects/stc && go test ./pkg/checker/... -count=1 -run TestUnused -v && go test ./pkg/checker/... -count=1 -run TestUnreachable -v</automated>
  </verify>
  <done>Unused variable detection correctly warns on unused local variables, skips interface variables (VAR_INPUT/OUTPUT/IN_OUT). Unreachable code detection warns on statements after RETURN/EXIT.</done>

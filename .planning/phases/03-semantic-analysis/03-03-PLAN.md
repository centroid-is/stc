  <action>
    Create `pkg/checker/check.go`:
    - Checker struct: table *symbols.Table, diags *diag.Collector, currentReturnType types.Type
    - NewChecker(table, diags) *Checker
    - CheckBodies(files []*ast.SourceFile) — walks all files, for each POU body:
      - Enter the POU's scope (look up via table.LookupPOU)
      - For each statement, call checkStmt
      - Exit scope

    - checkStmt(stmt ast.Statement):
      - AssignStmt: checkExpr(target) and checkExpr(value), verify value type CanWiden to target type. If target is from VAR_IN_OUT, verify value is an lvalue (Ident, IndexExpr, MemberAccessExpr — not Literal or BinaryExpr). Report SEMA001 for type mismatch, SEMA025 for VAR_IN_OUT literal.
      - IfStmt: checkExpr(condition) must be BOOL, recurse into Then/ElsIf/Else bodies
      - ForStmt: variable must be ANY_INT, From/To/By must be compatible integer types
      - WhileStmt: condition must be BOOL
      - RepeatStmt: condition must be BOOL
      - CaseStmt: checkExpr(expr), each branch label must be compatible type
      - CallStmt: resolve callee as FB instance, validate named args against FB's VAR_INPUT/VAR_OUTPUT
      - ReturnStmt: mark subsequent statements as unreachable (SEMA013)

    - checkExpr(expr ast.Expr) types.Type — returns resolved type:
      - Ident: lookup in scope chain, mark symbol Used, return its type. If not found, report SEMA010 (undeclared), return Invalid.
      - Literal: infer type from LitKind (LitInt->TypeDINT default, LitReal->TypeLREAL, LitBool->TypeBOOL, LitString->TypeSTRING, LitTime->TypeTIME, etc.). For LitTyped, use TypePrefix.
      - BinaryExpr: per RESEARCH.md checkBinaryExpr pattern — arithmetic ops use CommonType, comparison ops return BOOL, boolean ops require BOOL operands. Report SEMA003 for incompatible operator.
      - UnaryExpr: NOT requires BOOL, - requires ANY_NUM
      - CallExpr: resolve callee as Function, validate positional args count and types. For builtin functions with ANY_* params, use candidate resolution. Report SEMA020 (wrong count), SEMA021 (wrong type), SEMA022 (not callable).
      - MemberAccessExpr: checkExpr(object), if StructType look up member (SEMA024 if missing), if EnumType look up value, if FBType look up output/input
      - IndexExpr: checkExpr(object) must be ArrayType, indices must be ANY_INT, return element type. Report SEMA023 if not indexable.
      - DerefExpr: checkExpr(operand) must be PointerType, return base type (per user decision, represent but don't deeply validate)
      - ParenExpr: return checkExpr(inner)
      - ErrorNode: return Invalid (propagate, don't cascade)

    Create `pkg/checker/candidates.go`:
    - ResolveCandidates(fn *types.FunctionType, argTypes []types.Type) (types.Type, []types.Type, bool) — for functions with ANY_* generic params:
      1. Fill candidates: for each param with generic constraint, enumerate concrete types that match both the constraint AND the actual argument type
      2. Narrow candidates: intersect candidate sets, find single consistent assignment
      3. Return resolved return type and narrowed param types, or false if no valid assignment
    - Keep candidate sets bounded (max 16 per RESEARCH.md) to avoid explosion

    - TestComparisonReturnsBool: INT > INT returns BOOL
    - TestBooleanOpsRequireBool: TRUE AND FALSE ok, INT AND INT = error
    - TestFBInstanceCall: FB instance call with named args resolves correctly

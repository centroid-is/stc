  <action>
    **stdlib_math.go**: Create a StdlibFunctions map[string]func(args []Value) (Value, error) and register all math functions:
    - ABS: For int args, use manual abs (if neg, negate). For real args, use math.Abs.
    - SQRT, SIN, COS, TAN, ASIN, ACOS, ATAN, LN, LOG, EXP: Delegate to math.Sqrt, math.Sin, etc. Operate on Real values. Return ValReal.
    - EXPT: Always returns real via math.Pow. Per research open question, EXPT always returns REAL.
    - MIN, MAX: Compare two args. Handle both int and real. Return same kind as inputs.
    - LIMIT(MN, IN, MX): MAX(MN, MIN(IN, MX)). Handles int and real.
    - SEL(G, IN0, IN1): If G.Bool then IN1 else IN0.
    - MUX(K, IN0, IN1, ...): Select args[K.Int + 1] (offset by 1 since K is first arg). Bounds check.
    - MOVE(IN): Return IN unchanged.

    All functions take []Value and return (Value, error). The interpreter's evalCall will look up the function name in StdlibFunctions before falling through to user-defined functions.

    **stdlib_convert.go**: Add conversion functions to StdlibFunctions map. Pattern: function name *_TO_* maps to a conversion. Key implementations:
    - REAL_TO_INT, LREAL_TO_DINT: Use math.RoundToEven() for banker's rounding per IEC spec. Cast to int64.
    - INT_TO_REAL, DINT_TO_LREAL: Cast int64 to float64.
    - BOOL_TO_INT: false->0, true->1. INT_TO_BOOL: 0->false, nonzero->true.
    - INT_TO_STRING: strconv.FormatInt. STRING_TO_INT: strconv.ParseInt (return error on parse failure).
    - REAL_TO_STRING: strconv.FormatFloat with 'G' format. STRING_TO_REAL: strconv.ParseFloat.
    - BYTE_TO_INT, INT_TO_BYTE: Direct int64 cast with masking for BYTE (& 0xFF).
    - Generate the common combinations listed in pkg/types/builtin.go BuiltinFunctions conversion list.
    - For unsupported conversions, return a RuntimeError.
  </action>

  </verify>

<task type="auto" tdd="true">
  <name>Task 2: String functions with IEC 1-based indexing</name>
  <files>pkg/interp/stdlib_string.go, pkg/interp/stdlib_string_test.go</files>
  <behavior>
    - LEN("hello") -> 5, LEN("") -> 0
    - LEFT("hello", 3) -> "hel", LEFT("hi", 5) -> "hi"
    - RIGHT("hello", 3) -> "llo", RIGHT("hi", 5) -> "hi"
    - MID("hello", 3, 2) -> "ell" (3 chars starting at 1-based position 2)
    - MID("hello", 2, 1) -> "he" (2 chars starting at position 1)
    - MID("hello", 2, 10) -> "" (position beyond length returns empty)
    - CONCAT("hello", " world") -> "hello world"
    - FIND("hello world", "world") -> 7 (1-based position), FIND("hello", "xyz") -> 0
    - INSERT("hello", " world", 6) -> "hello world" (insert at 1-based position 6)
    - DELETE("hello world", 6, 6) -> "hello" (delete 6 chars at 1-based position 6)
    - REPLACE("hello world", "earth", 5, 7) -> "hello earth" (replace 5 chars at position 7 with "earth")
    - Edge: LEFT("hello", 0) -> "", RIGHT("hello", 0) -> "", MID("hello", 0, 1) -> ""
    - Edge: position < 1 returns empty/unchanged for MID/INSERT/DELETE/REPLACE
  </behavior>
  <action>
    **stdlib_string.go**: Add string functions to the StdlibFunctions map:

    - LEN(IN): len(IN.Str). Return ValInt.
    - LEFT(IN, L): IN.Str[:min(L, len(s))]. Return ValString.
    - RIGHT(IN, L): IN.Str[max(0, len(s)-L):]. Return ValString.
    - MID(IN, L, P): IEC 1-based. Convert P to 0-based: start = P-1. Extract start to start+L. Bounds-check. Per research: "L characters starting at position P (1-based)."
    - CONCAT(IN1, IN2): IN1.Str + IN2.Str. Return ValString.
    - FIND(IN1, IN2): strings.Index(IN1.Str, IN2.Str). Convert 0-based result to 1-based (+1). Return 0 if not found (strings.Index returns -1).
    - INSERT(IN1, IN2, P): Insert IN2 into IN1 at 1-based position P. Convert to 0-based: idx = P-1. s[:idx] + IN2 + s[idx:].
    - DELETE(IN, L, P): Delete L chars starting at 1-based position P. Convert: idx = P-1. s[:idx] + s[idx+L:].
    - REPLACE(IN1, IN2, L, P): Replace L chars at 1-based position P with IN2. s[:idx] + IN2 + s[idx+L:].

    Critical: ALL position parameters use IEC 1-based indexing. The conversion formula is always `goIdx = iecPos - 1`. Validate: if iecPos < 1, return empty/unchanged. If length exceeds bounds, truncate to available.

    Per user decision: Fixed-length strings backed by Go strings with length validation at boundaries. Add a helper `truncateString(s string, maxLen int) string` that enforces STRING[80] default max length. Apply truncation on assignment (in env.Set), not in function returns.
  </action>

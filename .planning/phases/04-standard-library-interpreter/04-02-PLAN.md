  <name>Task 1: StandardFB interface and FBInstance management</name>
  <files>pkg/interp/fb_instance.go, pkg/interp/fb_instance_test.go</files>
  <behavior>
    - StandardFB interface has Execute(dt time.Duration), SetInput(name string, v Value), GetOutput(name string) Value, GetInput(name string) Value methods
    - FBInstance struct wraps StandardFB (for stdlib FBs) or Env+Decl (for user-defined FBs)
    - FBInstance with StandardFB: SetInput sets on the FB, Execute delegates to FB, GetOutput reads from FB
    - FBInstance for user-defined FB: maintains Env with persistent state, Execute runs body statements
    - NewFBInstance("TON", stdlibFactory) creates instance with StandardFB
    - NewUserFBInstance(decl, parentEnv) creates instance with env initialized from VarBlocks
    - FBInstance state persists: call Execute twice, internal state accumulates
  </behavior>
  <action>
    Create pkg/interp/fb_instance.go:

    **StandardFB interface**: Execute(dt time.Duration), SetInput(name string, v Value), GetOutput(name string) Value, GetInput(name string) Value. This is the contract all stdlib FBs will implement in plan 04.

    **FBInstance struct**: TypeName string, FB StandardFB (non-nil for stdlib FBs), Env *Env (for user-defined FBs), Decl *ast.FunctionBlockDecl (AST for user-defined FBs). Methods: Execute(dt, interp), SetInput, GetOutput, GetInput that delegate to either the StandardFB or the Env depending on which is set.

    **StdlibFBFactory**: A map[string]func() StandardFB that plan 04 will populate. For now, initialize empty. The interpreter will look up FB type names in this factory when creating instances.

    **NewUserFBInstance(name string, decl *ast.FunctionBlockDecl, interp *Interpreter, parentEnv *Env) *FBInstance**: Creates an FBInstance with a new Env child of parentEnv. Walks decl.VarBlocks to define all variables with their zero/initial values in the env. This env persists across scan cycles.

    Update value.go: Change FBRef field from `any` to `*FBInstance` if it was declared as any. This allows member access (fb.Q, fb.ET) to resolve through the FBInstance.

    Wire into interpreter.go: When execStmt encounters a CallStmt whose callee resolves to an FBInstance value, call fbInstance.SetInput for each named arg, then fbInstance.Execute(dt), then copy output args back. When evalExpr encounters a MemberAccessExpr on an FBInstance value, call fbInstance.GetOutput(member).
  </action>

  </verify>

<task type="auto" tdd="true">
  <name>Task 2: Scan cycle engine with deterministic time and I/O table</name>
  <files>pkg/interp/scan.go, pkg/interp/scan_test.go</files>
  <behavior>
    - NewScanCycleEngine(program, symbolTable) creates engine with zero clock
    - SetInput("StartBtn", BoolValue(true)) stores input value
    - Tick(100ms) executes one scan cycle and advances clock by 100ms
    - GetOutput("MotorRunning") returns current output value after Tick
    - Clock() returns accumulated virtual time (deterministic, no wall-clock)
    - After 5 Ticks of 100ms each, Clock() returns 500ms
    - Inputs are copied into program env before body execution
    - Outputs are read from program env after body execution
    - Program VAR_INPUT variables receive values from SetInput
    - Program VAR_OUTPUT variables are readable via GetOutput
    - Calling Tick multiple times accumulates time correctly
    - FB instances within the program persist state across Ticks
  </behavior>
  <action>
    Create pkg/interp/scan.go:

    **ScanCycleEngine struct**: interp *Interpreter, inputs map[string]Value, outputs map[string]Value, clock time.Duration, program *ast.ProgramDecl, env *Env, initialized bool, inputNames []string (VAR_INPUT var names), outputNames []string (VAR_OUTPUT var names).

    **NewScanCycleEngine(program *ast.ProgramDecl, table *symbols.Table) *ScanCycleEngine**: Creates the engine. Walks program.VarBlocks to identify input/output variable names. Creates interpreter and root env. Initializes all program variables in the env with zero/initial values (only once, not per scan). Per user decision: deterministic time only, no wall-clock.

    **Tick(dt time.Duration) error**: (1) Copy inputs map into env for VAR_INPUT vars. (2) Set current dt on interpreter (so FB Execute calls receive it). (3) Call interp.execStatements(env, program.Body). (4) Copy VAR_OUTPUT vars from env into outputs map. (5) Advance clock += dt. Return any runtime error.

    **SetInput(name string, v Value)**: Store in inputs map (case-insensitive key). Error-safe: silently ignores unknown input names.

    **GetOutput(name string) Value**: Read from outputs map (case-insensitive key). Returns zero value if not found.

    **Clock() time.Duration**: Returns accumulated virtual time.

    **initializeEnv()**: Called once on first Tick (lazy init). Walk program.VarBlocks, for each VarDecl: determine IEC type from the symbol table, call Zero(type) or evaluate InitValue if present, define in env. Track which vars are inputs vs outputs based on VarBlock.Section.

    The interpreter needs a `dt` field (current scan delta) that gets passed to FB Execute calls. Add this field to Interpreter struct.

    Tests should use manually constructed ProgramDecl AST nodes with VarBlocks and simple Body statements. Test the full cycle: SetInput -> Tick -> GetOutput.
  </action>

SetInput/GetOutput provide programmatic I/O access. Program variables persist across scan cycles. FB instances within programs maintain state.</done>

MCP Go SDK API (modelcontextprotocol/go-sdk v1.4.1):
```go
server := mcp.NewServer(&mcp.Implementation{Name: "stc", Version: "1.0.0"}, nil)

type MyArgs struct {
    Code string `json:"code" jsonschema:"ST source code to parse"`
}

mcp.AddTool(server, &mcp.Tool{
    Name:        "stc_parse",
    Description: "Parse ST source and return AST with diagnostics",
}, func(ctx context.Context, req *mcp.CallToolRequest, args MyArgs) (*mcp.CallToolResult, any, error) {
    // handler
    return &mcp.CallToolResult{
        Content: []mcp.Content{&mcp.TextContent{Text: result}},
    }, nil, nil
})

server.Run(context.Background(), &mcp.StdioTransport{})
```

    Create cmd/stc-mcp/main.go:
    - Package main with func main()
    - Create mcp.NewServer with Implementation{Name: "stc-mcp", Version: "1.0.0"}
    - Call registerTools(server) to register all 6 tools

    Create cmd/stc-mcp/tools.go:
    - func registerTools(server *mcp.Server) that registers all 6 tools
    - Each tool uses mcp.AddTool with typed args struct and handler function

    Tool definitions (descriptions MUST be under 100 tokens each per MCP-07):

    1. stc_parse: Parse IEC 61131-3 ST source code and return AST with diagnostics.
       Args: { code: string (required), filename: string (optional, default "input.st") }
       Handler: Call parser.Parse(filename, code), marshal AST to JSON via ast.MarshalNode,
       return JSON object with "ast", "diagnostics", "has_errors" keys.

    2. stc_check: Type-check ST source code and return semantic diagnostics.
       Args: { code: string (required), vendor: string (optional, one of "beckhoff","schneider","portable") }
       Handler: Parse with parser.Parse, run analyzer.Analyze on parsed file.
       If vendor specified, set project.Config.Build.VendorTarget.
       Return JSON array of diagnostics.

    3. stc_test: Run ST unit tests in a directory.
       Args: { directory: string (required) }
       Handler: Call stctesting.Run(directory), format as JSON via stctesting.FormatJSON.
       Return the JSON string.

    4. stc_emit: Emit vendor-specific ST from source code.
       Args: { code: string (required), target: string (optional, default "portable", one of "beckhoff","schneider","portable") }
       Handler: Parse with parser.Parse, call emit.Emit with emit.Options{Target: emit.LookupTarget(target), Indent: "    ", UppercaseKeywords: true}.
       Return emitted code string.

    5. stc_lint: Lint ST source code for coding standard violations.
       Args: { code: string (required) }
       Handler: Parse with parser.Parse, call lint.LintFile with DefaultLintOptions().
       Return JSON array of diagnostics.

    6. stc_format: Format ST source code with consistent style.
       Args: { code: string (required) }
       Handler: Parse with parser.Parse, call format.Format with FormatOptions{Indent: "    ", UppercaseKeywords: true}.
       Return formatted code string.

    All handlers follow the same error pattern:
    - If the tool operation fails fatally, return error
    - Parse errors are NOT fatal -- include them in diagnostics and return the partial result
    - Return mcp.CallToolResult with TextContent containing JSON

    Create cmd/stc-mcp/tools_test.go:
    - Test each tool handler function directly (not through MCP transport)
    - Extract handler functions as package-level funcs so tests can call them
    - Use testify/assert for assertions
    - Test the description length constraint: split each description by whitespace, assert len <= 100

    Import the testing package as stctesting alias (same pattern as cmd/stc/test_cmd.go):

